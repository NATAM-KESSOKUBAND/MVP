# fonts_train — 폰트 분류기 학습용 폰트 폴더

여기에 **폰트 파일(.ttf / .otf / .ttc)** 을 넣고 학습하면, AI 폰트 분류기가
그 폰트로 렌더링된 자막을 영상에서 식별할 수 있게 됩니다.

## 핵심 원리

- 분류기는 **폰트 파일이 있는 폰트만** 학습할 수 있습니다.
  (폰트 모양을 직접 보고 배우기 때문 — 이름만으로는 학습 불가)
- 그래서 **격동고딕 같은 상업 폰트를 감지하려면 그 폰트 파일을 합법적으로
  보유**하고 있어야 합니다 (구매/라이선스 보유분).
- 무료 폰트도 함께 넣어야 합니다 → "이건 상업 폰트가 아님"을 구분하는 기준.

## 사용법

### 1. 폰트 파일 넣기
이 폴더(`fonts_train/`)에 `.ttf` / `.otf` 파일을 복사합니다.

### 2. manifest.json 작성
각 폰트가 상업용인지 표시합니다 (manifest에 없는 파일은 상업용으로 간주):

```json
[
  {"file": "GyeokdongGothic.otf", "name": "격동고딕",
   "commercial": true,  "foundry": "Sandoll", "risk": 0.88},
  {"file": "MyFreeFont.ttf",      "name": "무료폰트",
   "commercial": false, "foundry": "배포처"}
]
```

| 필드 | 의미 |
|------|------|
| `file` | 파일명 (이 폴더 기준) |
| `name` | 리포트에 표시될 폰트명 |
| `commercial` | `true` = 감지 대상(유료), `false` = 거절 클래스(무료) |
| `foundry` | 제작사/권리자 |
| `risk` | 위험도 0~1 (commercial=true일 때만 의미, 기본 0.80) |
| `index` | TTC 폰트의 인덱스 (보통 0, 생략 가능) |

### 3. 학습 실행
```
python tools/train_font_classifier.py
```
옵션: `--samples 400`(클래스당 샘플 수, 많을수록 정확/느림), `--epochs 10`

산출물: `models/font_classifier.pt` + `models/font_classifier_meta.json`
→ 다음 영상 분석부터 자동으로 사용됩니다 (재시작 불필요).

## 주의

- 시스템 내장 폰트(HY 상업 폰트 8종 + 무료/번들 12종)는 이 폴더가 비어 있어도
  자동으로 학습됩니다. 여기엔 **추가로 학습할 폰트만** 넣으면 됩니다.
- 폰트 1종을 추가할 때마다 **무료 폰트도 충분히 섞여 있어야** 오탐이 줄어듭니다
  (상업 폰트만 잔뜩 학습하면 모든 자막을 상업 폰트로 오인).
- `예시-*.otf` 항목은 샘플이므로 실제 파일이 없으면 자동 스킵됩니다.
